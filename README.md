# 🧾 Controle Loja

Projeto focado na evolução de um sistema de controle operacional e financeiro, partindo de uma planilha simples até um sistema completo com interface.
Esse projeto surgiu de uma necessidade real: controlar pedidos e dividir o lucro com minha irmã com precisão.

A primeira versão foi construída de forma orgânica, com fórmulas sendo adicionadas conforme novas necessidades apareciam. Apesar de funcional para uso pessoal, a estrutura ficou difícil de manter e pouco intuitiva para outras pessoas.

A versão 2 nasceu com o objetivo de evoluir esse modelo, criando uma estrutura mais organizada, com separação entre entrada de dados e processamento, além de melhorias na usabilidade e visualização das informações.

---

## 🚀 Etapas do Projeto

### 🔹 Etapa 1 — Base Manual (v1) ✅ Concluída

Planilha inicial criada para centralizar o controle da operação.

**Funcionalidades:**

* Registro de pedidos
* Controle de faturamento
* Fluxo de caixa
* Follow-up e ocorrências
* Controle de reembolsos
* Análise de itens

**Limitação:**

* Inputs e cálculos misturados
* Alto risco de erro ao editar fórmulas
* Estrutura pouco escalável

---

### 🔹 Etapa 2 — Sistema Automatizado (v2) 🔄 Em desenvolvimento

Reestruturação da planilha com foco em automação, usabilidade e segurança.

**O que já foi feito:**

* Separação entre input e processamento
* Criação do motor de dados (Banco de Dados)
* Automação de cálculos e indicadores
* Redução de risco de quebra de fórmulas

**O que está sendo desenvolvido agora:**

* Layout de input mais limpo e intuitivo
* Redução da quantidade de abas
* Campo de busca de pedidos
* Substituição de tabelas dinâmicas por dashboards
* Organização visual para facilitar uso no dia a dia

**Objetivo da etapa:**
Transformar a planilha em um sistema mais próximo de uma aplicação, mesmo ainda dentro do Excel.

### ⚙️ Lógica automatizada com fórmulas

A planilha utiliza fórmulas dinâmicas (MAP, PROCX, INDIRETO) para se autoalimentar e aplicar regras de negócio automaticamente.

---

### 🔹 Etapa 3 — Versão em Python (planejado)

Migração da lógica para código.

**Objetivo:**

* Recriar o motor de dados em Python
* Melhorar performance e controle
* Automatizar ainda mais o processamento

---

### 🔹 Etapa 4 — Sistema com Interface (planejado)

Construção de uma aplicação completa.

**Objetivo:**

* Interface amigável (tipo sistema mesmo)
* Entrada de dados estruturada
* Dashboards interativos
* Gestão completa da operação

---

## 🧠 Conceito do Projeto

Evolução prática de um sistema real:

> Planilha manual → Planilha automatizada → Código → Sistema completo

Foco em:

* Automação
* Redução de erro humano
* Organização de dados
* Escalabilidade
* Experiência do usuário
